MUNSHI IMPLEMENTATION LOG
=========================

This file records what is currently implemented in the repository, not only
what was planned. The original problem statement is in ps.txt and the product
vision is in solution.txt.

SCOPE
-----

The current build includes:

- backend/ as the FastAPI compliance brain and API layer.
- twilio/ as the WhatsApp channel adapter.
- frontendmain/ as the active Vite React frontend.
- A CA admin dashboard mode.
- A client self-service portal mode.
- MongoDB persistence for trader data, CA platform data, sessions, issues,
  audit logs, alerts, reminders, suppliers, and ERP keys.

The older frontend/ directory is legacy for current docs. The active website
for this build is frontendmain/.

ARCHITECTURE
------------

backend/
  FastAPI app mounted from backend.main. It exposes the original trader
  pipeline, the CA platform under /ca, the client portal under /client, a
  /health check, rate limiting, CORS, and startup Mongo index setup.

MongoDB
  Stores WhatsApp/trader records plus CA platform collections. CA platform
  collections are prefixed to avoid shape collisions with trader records.

frontendmain/
  Vite React SPA. It has Admin Mode and Client Mode. The API client stores
  separate CA and client bearer tokens so the scopes do not overwrite each
  other.

twilio/
  Node/Express server. It owns the Twilio webhook, conversation state,
  media download, voice transcription, language switching, WhatsApp replies,
  proactive nudges, and optional generated voice replies. GST business logic
  is delegated to backend/.

AI/deterministic split
  Gemini and Groq are used for extraction, transcription, narration, and
  optional supplier-message polish. Matching, issue classification, ITC
  arithmetic, HSN/SAC validation, timeline, simulator, and PDF numbers are
  deterministic backend code. AI does not compute rupee figures.

IMPLEMENTED BACKEND FEATURES
----------------------------

Core trader/compliance pipeline:

- Invoice extraction endpoint for image/PDF invoice inputs.
- Extraction confidence and review indicators.
- GSTR-2B ingestion from CSV/XLSX, with PDF parsing support in the parser.
- Confirmed invoice storage.
- Deterministic reconciliation by GSTIN, invoice number, amount, tax, HSN,
  duplicates, and missing records.
- ITC impact calculation.
- HSN/SAC lookup and categorisation, including service/product category
  support from local master data.
- Anomaly detection and recommendation generation.
- Health check that reports Mongo, Gemini, and Groq readiness.
- Rate limiting middleware.

CA platform APIs:

- CA login and logout.
- Static CA bearer-token support for scripts/service calls.
- Firm portfolio summary.
- Client listing, creation, read, soft delete, and export.
- Client portal account creation.
- Invoice list/add/upload endpoints.
- GSTR-2B baseline add/upload endpoints.
- Reconciliation get/run/history endpoints.
- All-issue and per-client issue lists.
- Issue summary and status update.
- Supplier-message draft generation and optional Groq polish.
- ERP invoice ingest endpoint authenticated with per-client ERP API key.
- ERP import profiles and column mapping for common accounting exports.
- ERP sync log.

Client portal APIs:

- Client login/logout.
- /client/me scoped to the logged-in client.
- Client invoice upload/list.
- Client GSTR-2B upload.
- Client reconciliation get/run/history.
- Client issue list/status update.
- Client supplier-message draft.
- Client vendors, timeline, simulator, and monthly PDF report.

Security/privacy/data-management:

- CA sessions and client sessions with hashed bearer tokens.
- Human passwords stored with bcrypt.
- High-entropy tokens and ERP API keys hashed at rest with a server-side
  pepper.
- ERP API key raw value is only shown at creation.
- PII masking helpers for GSTIN/phone where needed.
- Soft delete for clients.
- Client data export.
- Activity/audit logs.
- Retention purge for old audit logs.
- Optional service token for trusted service-to-service calls.

Analytics and workflow:

- Multi-period reconciliation history.
- ITC recovery tracker through issue statuses.
- Action cards/recommendations.
- Vendor compliance scorecard.
- Supplier risk forecasting.
- Alerts and acknowledgements.
- Due-reminder calculation.
- GST compliance timeline.
- Monthly PDF report generation.
- Financial impact simulator.
- CA knowledge graph views for supplier and client relationships.

IMPLEMENTED TWILIO FEATURES
---------------------------

- Express server with /health, /sms webhook, /nudge, and temporary /media
  routes for generated audio.
- Mongo-backed per-phone session state.
- English, Hindi, and Marathi language handling.
- Text commands for hi/help/menu, language switching, check/reconcile,
  status, remind, and issue detail selection.
- WhatsApp voice-note input through Groq transcription.
- Invoice image/PDF upload flow.
- Batch invoice media handling.
- Confirmation and correction handling for extracted invoice fields.
- Review queue hooks.
- GSTR-2B spreadsheet upload flow.
- PDF media disambiguation between invoice and GSTR-2B.
- Reconciliation trigger and interactive issue list.
- Supplier reminder draft reply.
- Q&A/help-style fallback using intent parsing.
- Proactive nudge job protected by NUDGE_SECRET.
- Optional TTS voice reply layer behind ENABLE_TTS; text remains canonical.
